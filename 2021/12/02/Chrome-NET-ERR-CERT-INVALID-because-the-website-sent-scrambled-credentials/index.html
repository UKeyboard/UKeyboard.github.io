解决Chrome NET::ERR_CERT_INVALID because the website sent scrambled credentials问题的神奇密码

问题描述

昨天服务器的证书更新后，客户端突然就不能访问了，Chrome直接报类似下面这样的错误：

You cannot visit xxxxxx right now because the website sent scrambled credentials that Google Chrome cannot process. Network errors and attacks are usually temporary, so this page will probably work later.

具体情况如下图所示：

通常出现这种状况Chrome会在页面折叠起来的“高级”或”Advance”里提供一个可以访问的连接，尽管存在风险。但是，这次没有任何可以点击的连接，只有 NET::ERR_CERT_INVALID错误。

解决方案

从页面提供的信息来看是因为证书的变更触发了Chrome的安全机制，出于安全考虑Chrome阻断了用户请求，所以可以通过本地设置来规避这个问题。

网上提供的方式是：激活当前报错页面，在页面任意位置输入thisisunsafe或badidea即可解除Chrome的安全警报。个人觉得这种解决方式还挺有意思的，一般人发现不了Google在这里设置了这样的神奇密码。。。。

最终，我通过在警告页面上键入thisisunsafe顺利进入页面，输入该密码后页面会自动刷新。另一个密码badidea下次遇到类似问题时再尝试吧。
